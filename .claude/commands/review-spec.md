# Review Spec

Use this workflow before implementation.

Check the target `SPEC.md` for:

- Ambiguous goals or missing non-goals.
- Uncited system facts.
- Hidden architecture decisions.
- Missing safety invariants.
- Missing or non-runnable acceptance commands.
- Missing rollback plan.
- Open questions that block execution.
- Scope that conflicts with `AGENTS.md` or project docs.

Return findings first, ordered by severity. If the spec is executable, say so
clearly and list any residual risk.
