# Verify Spec

Use this workflow after implementation.

1. Inspect the diff against the approved spec.
2. Confirm no unrelated files changed.
3. Run all acceptance commands listed in the spec.
4. Run the repo's normal verification gate if different.
5. Check docs and instructions for consistency.
6. Report exact command results, residual risk, and Mimir memory candidates.

Do not say complete unless verification ran in the current session.
